Overview

Open source software and software supply chains are increasingly vulnerable to new attacks globally. To address these challenges, the public and private sectors have to collaborate and co-develop new processes, tools, and initiatives.

Learn how the coupling of Supply-chain Levels for Software Artifacts (SLSA) and Software Bill of Materials (SBOM) can improve software supply chain security and reduce incidents. And discover common Application Security (AppSec) challenges that can be prevented through good security practices.

You will also gain insights on concrete steps that can be taken to build a strong foundation for a more secure future.

Who should attend: Cybersecurity professionals, software developers, and engineers; Levelling: Open to all
